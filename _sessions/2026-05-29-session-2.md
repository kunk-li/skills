## 关键技术决策

### 为什么用 args 数组而不是 shell 命令字符串
schema 文档原话:"Path placeholders like ${CLAUDE_PLUGIN_ROOT} are substituted per-element as plain strings, so paths with quotes, $, or backticks never reach a shell parser."

跨平台/跨 shell 最稳。Python 路径含 `:\` 和 `\` 转义在 bash/PowerShell 都是雷区。

### 为什么硬编码 `C:\\ProgramData\\miniconda3\\python.exe`
user 这台机器 `python` 没在 PATH(WindowsApps stub 拦截)。MVP 阶段先硬编码,未来跨机器再用 `${PYTHON}` 或加 PATH。

### 为什么 SessionStart hook 用 `additionalContext` 字段
hook 返回 JSON `{"hookSpecificOutput": {"hookEventName": "SessionStart", "additionalContext": "..."}}` 时,内容会被 Claude Code 注入为 session 启动上下文——比 systemMessage(用户可见)更适合"项目记忆"用途。

### 为什么不 SessionEnd 而是 Stop
SessionEnd 在 session 真正结束时触发,那时 Claude 已经停了,没法再让它写文件。要让 Claude 自己写 summary,只能在 Stop 时 asyncRewake 把 Claude 叫起来——但每次回答完都触发,有循环风险。

**取舍**:本 session 决定**先用 CLAUDE.md 里的明确指令 + SessionStart 注入提醒**,让 Claude 在 session 内主动更新 STATUS.md + 写 session 总结。**自动 hook 留到 Phase 1 中后期再做**——现在加风险大于收益。
