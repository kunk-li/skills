Skills 流程优化方案
v2 · Reactive Edition From Static DAG → Reactive Event-Driven System

在 v1 流程基础上引入 事件总线、产物契约、渐进门禁、反馈回流、熔断保护、分层路由、SLA 人工队列 七项核心改造,把"被动执行的 DAG"升级为"主动感知的反应式系统"。

7核心优化项

Seven Optimizations · 七项优化

改造项一览

P0
// 优化 02

📜 产物契约层

从字符串文件名升级为带 Schema、版本号、生产者-消费者元数据的契约对象。改字段必升版本,下游可感知。

P0
// 优化 06

🚦 渐进式门禁 (Shift-Left)

把检查能力下沉到每个节点(节点自检),N100/N260/N380 退化为汇总判定。问题在节点内修复,不流转下游。

P1
// 优化 01

⚡ 事件总线

横切层(文档/任务/治理)从 DAG 末尾节点改为事件订阅者,主流程任何节点完成即触发,文档不再滞后于代码。

P1
// 优化 04

🔄 线上反馈回流

新增 N005 节点聚合监控/复盘/缺陷/NPS,反推数据驱动的需求假设,直接喂给 N010 与 N060。

P2
// 优化 03

🎯 三层分层路由

N370 路由从 130+ 全集选择,改为 阶段→节点→节点内 三层决策,每层 ≤11 选 1,准确率显著提升。

P2
// 优化 07

👥 人工确认 SLA 队列

每个待确认项打 P0/P1/P2 优先级 + SLA 时限 + 风险标注 + 推荐动作。超时自动升级,人不再被淹没。

P3
// 优化 05

⚡ 回环熔断器

所有回环边(热修复/质量升级)增加 max_iterations + cooldown + 自动升级机制,防止"修一次坏一次"死循环。
